Omokai

status: functional end-to-end

Solo developer · technical assignment

Closes the loop from spoken intent to physical robot motion. A natural-language instruction is parsed by an LLM into a structured mission, checked against a deterministic safety validator, and executed as a real Nav2 navigation task inside Gazebo. The goal wasn't just to make an LLM output JSON, it was to build the guardrails that make it safe to act on.

key decision — put a hard validation layer between the LLM and the executor. The LLM is treated as untrusted input: nothing reaches Nav2 without first passing schema checks and safety bounds, so a hallucinated waypoint fails loudly before it ever reaches a motor command.

XLR8

status: voice pipeline complete

Solo developer · voice-driven robotics assistant

A modular voice-assistant system built on ROS 2, where independent nodes handle speech recognition, intent classification, LLM-based reasoning, and speech synthesis, communicating entirely through ROS topics, services, and actions. Designed as the interaction layer for a robotics platform, with a defined interface for downstream robot-action execution.

key decision — implemented speech output as a cancellable ROS 2 action rather than a fire-and-forget service, enabling real-time barge-in: the system stops talking the instant the user starts speaking again.

DAUS

status: published

Author · published research paper

A modular, cost-efficient autonomous drone system for disaster response. The UAS autonomously scans a user-defined area, detects disaster zones with a YOLOv8-based vision model, and performs payload drops without ground control, fusing LiDAR, PX4 telemetry, and GPS onboard for precise delivery. Validated through SITL simulation and field testing: 2 kg payload capacity, 25-minute flight endurance.

Quadruped Robot

status: earlier project

Solo developer · personal project

A 12-DOF quadruped robot built from the ground up on an Arduino Nano and twelve SG90 servo motors, with coordinated gait control and motion sequencing for walking and expressive movements.

key decision — used predefined gait sequences with synchronized joint trajectories, prioritizing repeatable locomotion over computationally intensive approaches like inverse kinematics, to stay within the Arduino Nano's headroom.
